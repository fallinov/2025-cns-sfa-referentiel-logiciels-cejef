# Accessibilité — WCAG 2.2 (résumé)

- Respecter les critères de contraste (au moins WCAG AA)  
- Utiliser des tailles de texte lisibles  
- Permettre la navigation au clavier  
- Fournir une structure sémantique HTML correcte (`<main>`, `<nav>`, `<header>`, etc.)  
- Éviter de transmettre l’information uniquement par la couleur  
- Assurer un focus visible et un ordre de tabulation logique sur tous les éléments interactifs
- Pour l’UX globale, croiser cette checklist avec `accessibility/ux-checklist.md`
