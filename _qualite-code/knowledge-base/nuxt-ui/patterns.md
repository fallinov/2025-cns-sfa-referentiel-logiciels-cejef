# Nuxt UI — Patterns recommandés

- Favoriser les composants existants plutôt que recréer de zéro  
- Utiliser les slots pour personnaliser sans casser le comportement natif  
- Ajouter seulement une surcouche minimale de style (via classes ou Tailwind)  
- Éviter le CSS custom complexe sauf besoin réel expliqué  
- S’appuyer sur les états intégrés (`loading`, `disabled`, `error`) plutôt que re-coder la logique
