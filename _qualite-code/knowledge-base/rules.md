# Règles globales d’écriture

- Appliquer sécurité avant tout
- Respecter l’accessibilité (WCAG)
- Favoriser maintenabilité, clarté, modularité
- Utiliser Nuxt, Nuxt UI, Tailwind avant de réinventer
- Éviter code mort, duplication, CSS/JS custom non justifié
- Priorité des règles : 1) Sécurité 2) Accessibilité & UX 3) Maintenabilité & Architecture 4) Performance 5) Style & esthétique
- Vérifier les checklists (WCAG/ARIA, UX) avant de valider un écran ou un flow
