# Tailwind CSS — Bonnes pratiques

- Utiliser les classes utilitaires plutôt que du CSS custom  
- Ne pas dupliquer du CSS déjà possible via Tailwind  
- Respecter la configuration de design system (couleurs, typo, spacing)  
- Factoriser des composants si des ensembles de classes reviennent souvent  
- Garder une approche mobile-first et limiter le nombre de variantes superflues
